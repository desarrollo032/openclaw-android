## 19. 2차 블로커 점검

**Aidan**: "블로커 한번 더 살펴봐."

→ 에이전트가 더 깊이 분석하여 4개 추가 발견:

5. **com.termux 문자열 구분**: Java 패키지, 파일 경로, 퍼미션, 환경변수 등 맥락별로 치환 여부가 다름. 단순 전체 치환하면 고장.

6. **IntentFilter가 explicit Intent에서 동작 안 함**: 블로커 3의 해결책(IntentFilter alias)이 기술적으로 불가능. CLI 스크립트가 `am broadcast -n com.termux.api/.SomeReceiver`로 explicit Intent 사용. 다른 패키지명 앱은 수신 불가.

## 23. pkg upgrade 패치 깨짐 문제

**Aidan**: "내가 뭔가 놓친 것이 있을까?"

**Aidan**: "네 의견대로 한다. 그런데, v1 에서도 비슷한 문제가 있었던거 같아. 뭔가를 업데이트 하면 우리 제공된 패치가 깨지는거."

## 24. 패치 깨짐 전수 점검

**Aidan**: "이런 식으로 패치 깨지는걸 방어해야 하는 것이 또 있어?"

→ 에이전트: `pkg upgrade`에 의해 깨질 수 있는 모든 지점 점검.
- termux-api CLI: wrapper로 방어 완료
- apt/dpkg 설정 파일: `pkg upgrade`로 덮어씌워질 가능성 있음
- glibc-runner: pacman 관리, apt와 독립이므로 영향 없음
- 앱 Java 코드: pkg upgrade 범위 밖이므로 영향 없음

추가 발견: termux-api에 새 명령어가 추가되면 wrapper가 없음. 개별 wrapper 대신 **단일 디스패처 스크립트 + symlink** 방식 제안.
